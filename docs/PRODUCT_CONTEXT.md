# PRODUCT_CONTEXT · сервис «Лена»

Документ задаёт **единый продуктовый контекст** для разработки, сопровождения и для агентов (в т.ч. в Cursor): что такое Лена, какие цели у tender-prep и что считается вне зоны ответственности.

## Проблема

Участник закупок тратит много времени на чтение положений, технических заданий и форм: нужно вытащить обязательные требования, сроки, критерии оценки и сопоставить их с возможностями поставщика без пропусков и противоречий.

## Решение

**«Лена»** — цифровой помощник по подготовке тендерной заявки:

- помогает структурировать требования из пакета документов закупки;
- поддерживает матрицу соответствия (требование → доказательство/раздел заявки);
- генерирует черновики текстов под формат заказчика, с явными пометками, где нужна проверка юриста или эксперта.

**tender-prep** — репозиторий, где фиксируются:

- продуктовые допущения и глоссарий;
- договорённости по интеграции с **parserit** (см. [PARSERIT_INTEGRATION.md](PARSERIT_INTEGRATION.md));
- инструкции для среды разработки, в частности **Git на Windows** ([WINDOWS_GIT.md](WINDOWS_GIT.md)).

## Пользователи и сценарии

| Роль | Типовой сценарий |
|------|------------------|
| Менеджер по тендерам | Загрузка комплекта, запуск парсинга, проверка матрицы, выгрузка в шаблон заявки |
| Технический специалист | Настройка связки с Windmill, обновление путей к флоу, контроль версий промптов/правил |
| ИИ-агент в IDE | Чтение этого файла и смежных `docs/*.md` перед правками кода или генерацией ответов по тендеру |

## Ключевые сущности

- **Закупка** — логическая единица: извещение, документация, формы, возможные разъяснения.
- **Лот / предмет** — то, на что подаётся заявка в рамках закупки.
- **Требование** — формулировка из документации, которую нужно закрыть документом или текстом в заявке.
- **Матрица соответствия** — таблица «требование → статус → ссылка на приложение/раздел».
- **Результат parserit** — структурированный вывод пайплайна Windmill (JSON/таблица), который Лена или оператор используют как основу, а не как юридически заверенный факт.

## Границы ответственности

**В зоне Лены и tender-prep**

- Подготовка структуры и черновиков под конкретную закупку.
- Согласованные вызовы parserit и обработка ожидаемых форматов ответа.

**Вне зоны (явная эскалация человеку)**

- Юридическая квалификация рисков, подписание заявки, решение об участии.
- Окончательная проверка соответствия законодательству и регламенту площадки.
- Любые действия от имени организации на ЭТП без явного поручения.

## Принципы для агентов

1. Не выдумывать требования: опираться на текст закупки или на помеченный вывод parserit.
2. Помечать неуверенность и ссылаться на источник (файл, страница, пункт).
3. Соблюдать контракт интеграции с parserit: имена полей, пути флоу, обработка ошибок и таймаутов.
4. При работе в Windows учитывать особенности Git из [WINDOWS_GIT.md](WINDOWS_GIT.md), чтобы не портить диффы и кодировки.
