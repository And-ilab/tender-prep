# Стратегия контекста для Лены

Краткая фиксация концепции: **что хранить где** и **в каком порядке** подмешивать в работу агента. Детальная настройка Drive и CLI — в [GOOGLE_DRIVE.md](GOOGLE_DRIVE.md) и [CORPUS_AND_RAG.md](CORPUS_AND_RAG.md).

## Два слоя смысла

### 1. Долгая память (архив накопленных знаний)

- **Что это:** ранее поданные тендеры, типовые документы, большой корпус по годам/проектам.
- **Задача:** справочник и карта материалов, а не «весь текст в системный промпт».
- **Как в продукте:** папка-архив на Google Drive (например «2025») + при необходимости индекс (`archive-context-build` → Markdown со ссылками в `_lena/context`) или выборочная выгрузка / RAG ([CORPUS_AND_RAG.md](CORPUS_AND_RAG.md)).

### 2. Директивы и советы (обучение, политика поведения)

- **Что это:** короткие правила: стиль ответа, чек-листы, «как отвечать на типовой вопрос».
- **Задача:** быстро менять поведение без пересборки всего архива.
- **Как хранить (план):** отдельно от архива — например маленькие `.md` в `_lena/library` или отдельная папка вроде `_lena/directives` (если заведёте по соглашению в команде).
- **Обновление:** редактирование файлов на Drive или версия в git для технических правок.
- **База в репозитории:** общие правила поведения Лены зафиксированы в [LENA_RULES.md](LENA_RULES.md); при необходимости дублируйте или сжимайте их в файлы на Drive под конкретную команду.

## Порядок приоритета в рантайме (рекомендация)

1. **Системный промпт** (роль, формат, жёсткие запреты) — в коде/конфиге, меняется редко.
2. **Директивы** — мало текста, «как работать сейчас».
3. **Материалы текущего тендера** — `inputs`, заметки, черновики.
4. **Архив** — по необходимости: индекс, ссылки, фрагменты после поиска/RAG, а не целиком.

Так архив не перетирается советами, а советы не раздувают контекст до нечитаемого размера.

## Дообучение модели vs настройка продукта

- **Fine-tune / дообучение весов** — отдельный пайплайн: датасет из пар «ситуация → эталон»; директивы могут быть источником примеров для датасета.
- **Итерация без переобучения:** обновление файлов директив + ревью ответов Лены; в проде эти файлы подмешиваются в `agent-bundle` / список контекста на Drive — этого достаточно для большинства этапов «обучения» продукта.

## Связь с переменными окружения

- **`LENA_EXTRA_CONTEXT_FOLDERS`** — дополнительные **корни** папок на Drive, файлы **только из корня** каждой папки попадают в `context-list` и бандл (вложенные подпапки не обходятся). Для глубокого архива удобнее **индекс в `_lena/context`**, а не полагаться на extra-корень без уплощения структуры.

Дальнейшая донастройка директивного слоя — после закрытия настройки долгой памяти по чеклисту в [GOOGLE_DRIVE.md](GOOGLE_DRIVE.md#lena-long-memory-archive).
