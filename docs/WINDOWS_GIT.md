# Git на Windows для tender-prep и сервиса «Лена»

Краткие рекомендации для разработчиков и агентов, которые коммитят из Windows в этот репозиторий, чтобы не ломать диффы, кодировки и линковку с CI.

## Установка

1. Установите [Git for Windows](https://git-scm.com/download/win) (включает Git Bash).
2. При необходимости — [Git LFS](https://git-lfs.com/) для больших вложений (PDF пакетов закупок, если команда решит хранить их в Git).

## Обязательные настройки репозитория

Выполните один раз после клонирования (пути можно скорректировать под вашу среду):

```bash
git config core.autocrlf false
git config core.eol lf
git config core.longpaths true
```

**Почему**

- `autocrlf false` + `eol lf` — единые переводы строк LF, как на Linux и в большинстве CI; меньше шума в диффах для `*.md`, `*.json`, `*.ts` и т.д.
- `longpaths true` — длинные пути Windows не обрывают операции Git.

Глобально те же значения допустимы, если у вас нет противоречащих политик в других проектах:

```bash
git config --global core.autocrlf false
git config --global core.eol lf
git config --global core.longpaths true
```

## Кодировка и PowerShell

- Сохраняйте текстовые файлы в **UTF-8** (в VS Code / Cursor: «UTF-8» в статус-баре).
- В PowerShell 7+ по умолчанию удобнее работа с Unicode; в старых консолях избегайте смешения OEM и UTF-8 при ручном `type`/`copy`.

## Учётные данные

- **HTTPS**: используйте диспетчер учётных данных Windows или менеджер от Git for Windows; не кладите пароли в URL remote.
- **SSH**: ключи `~/.ssh`, агент `ssh-agent`; remote вида `git@github.com:org/tender-prep.git`.

## Безопасность

- Не коммитьте токены Windmill, пароли ЭТП, персональные данные контрагентов.
- Для локальных URL и секретов используйте `.env` вне Git (и добавьте `.env` в `.gitignore`, когда появится кодовая база).

## Клонирование в нестандартный каталог

Если Git ругается на `safe.directory`, добавьте корень репозитория:

```bash
git config --global --add safe.directory C:/tender-prep
```

## Связь с документацией Лены

Правки в `docs/*.md` и промптах должны проходить тем же Git-потоком, что и код Лены: маленькие атомарные коммиты, понятные сообщения на русском или английском — по договорённости команды, но **единообразно внутри репозитория**.
