# Пошагово: Google Cloud → Drive API → ключ → Node (tender-prep)

Ниже — включение **Google Drive API**, создание **сервисного аккаунта** и **JSON-ключа**, выдача доступа к папке на Диске и проверка из этого репозитория. Интерфейс Google Cloud иногда меняется; названия пунктов даны **как в англоязычной консоли** (часто так и показывается), рядом — что искать по смыслу.

---

## Шаг 0. Что в итоге получится

- В Google Cloud есть **проект**, в нём включён **Drive API**.
- Есть **сервисный аккаунт** (робот-учётная запись) и файл **`*.json`** с закрытым ключом.
- На Google Drive есть **корневая папка**, расшаренная этому роботу.
- На вашем ПК в переменной окружения указан **путь к JSON**; команда `node src/cli.js drive …` ходит в **Google Drive API по HTTPS** с этим ключом (через библиотеку `googleapis` в Node).

Сервисный аккаунт **не равен** вашему личному Google-аккаунту: у него свой e-mail `…@…iam.gserviceaccount.com`. Он видит только то, что вы ему **явно откроете** (шаринг папки или доступ на Shared Drive).

---

## Шаг 1. Войти в Google Cloud Console

1. Откройте [https://console.cloud.google.com/](https://console.cloud.google.com/).
2. Войдите под учётной записью Google, которой вы будете владеть проектом (часто — рабочий аккаунт).

---

## Шаг 2. Создать или выбрать проект

1. Вверху рядом с логотипом Google Cloud нажмите на **название проекта** (Project).
2. Нажмите **New Project** / «Новый проект», если нужен отдельный проект под тендеры (например `tender-prep-drive`).
3. Укажите имя, при необходимости организацию, создайте проект и **переключитесь** на него в том же селекторе.

*Если проект уже есть — просто выберите его.*

---

## Шаг 3. Включить биллинг (если консоль потребует)

Для многих API Google просит привязать **Billing account** (платёжный аккаунт). Для обычных вызовов Drive API в разумных объёмах часто хватает **бесплатного лимита**, но привязка карты/договора в консоли может быть обязательной политикой организации.

1. Меню **☰** → **Billing** / «Оплата».
2. Свяжите проект с платёжным аккаунтом, если мастер это предлагает.

Если ваш ИБ-отдел запрещает биллинг — согласуйте с ними; без этого шаг «Enable API» иногда блокируется.

---

## Шаг 4. Включить Google Drive API

1. Меню **☰** → **APIs & Services** → **Library** (или «API и сервисы» → «Библиотека»).
2. В поиске введите **Google Drive API**.
3. Откройте карточку **Google Drive API**.
4. Нажмите **Enable** / «Включить».

Пока API не включён, запросы из Node получат ошибки вида «API not enabled».

---

## Шаг 5. Создать сервисный аккаунт

1. Меню **☰** → **IAM & Admin** → **Service Accounts** («Учётные записи сервисов»).
2. **+ Create service account** / «Создать учётную запись сервиса».
3. Заполните:
   - **Service account name** — например `lena-drive`;
   - **Service account ID** — подставится автоматически;
   - **Description** — по желанию («Доступ к папке тендеров на Drive»).
4. Нажмите **Create and continue**.
5. Роль на уровне проекта (**Grant this service account access to the project**):
   - для **минимальной** схемы часто достаточно не добавлять роли здесь вообще (доступ к **конкретной папке** на Диске выдадите шагом 9 через «Поделиться»);
   - если политика организации требует роль: можно оставить пустым или уточнить у админа (не путать с правами на **Drive** — их даёт шаринг папки).
6. **Done** / «Готово».

---

## Шаг 6. Выпустить JSON-ключ

1. В списке **Service Accounts** нажмите на созданную строку (email вида `lena-drive@….iam.gserviceaccount.com`).
2. Вкладка **Keys** / «Ключи».
3. **Add key** → **Create new key** → тип **JSON** → **Create**.
4. Браузер скачает файл `….json`. **Сохраните его в безопасное место** (например `C:\secrets\lena-drive-sa.json`), **не коммитьте** в Git.

Внутри JSON есть поле **`client_email`** — это адрес «робота». Он понадобится для шаринга папки на Диске.

---

## Шаг 7. (Опционально) Ограничить ключ

По политике ИБ иногда:

- запрещают долгоживущие JSON-ключи → используют **Secret Manager**, CI-секреты и т.д.;
- требуют ротацию ключей.

Для локального теста достаточно одного JSON; для прода — по регламенту компании.

---

## Шаг 8. Папка на Google Drive

1. Откройте [Google Drive](https://drive.google.com/) под **своим** пользователем.
2. Создайте папку, например `Тендеры — тест Lena`.
3. Откройте папку и скопируйте **ссылку** из адресной строки (в ней будет идентификатор папки после `/folders/`).

Эта папка — **корень** для команд `drive workspace-ensure` и т.д. в нашем репозитории.

---

## Шаг 9. Поделиться папкой с сервисным аккаунтом

1. В Google Drive: правая кнопка по папке → **Share** / «Настройки доступа».
2. В поле людей вставьте **`client_email`** из JSON (строка вида `…@…iam.gserviceaccount.com`).
3. Роль: **Editor** / «Редактор» (нужно, чтобы `workspace-ensure`, загрузка и копирование шаблонов работали).
4. Снимите галочку «Уведомить», если не хотите письмо роботу; **Send** / «Готово».

Без этого шага любые вызовы API от имени сервисного аккаунта **не увидят** содержимое вашей личной папки.

---

## Шаг 10. Переменная окружения с путём к ключу

Отдельный **`npm install` для Drive не нужен** — в репозитории вызовы Drive API идут через встроенный `fetch` и JWT (см. `src/drive/authToken.js`).

В PowerShell (пример пути под Windows):

```powershell
$env:GOOGLE_DRIVE_CREDENTIALS = "C:\secrets\lena-drive-sa.json"
```

Или задайте **`GOOGLE_APPLICATION_CREDENTIALS`** тем же путём — код в репозитории принимает оба варианта (см. `src/drive/config.js`).

---

## Шаг 11. Проверка из Node (этот репозиторий)

```powershell
node src/cli.js drive workspace-ensure "https://drive.google.com/drive/folders/ВАШ_ID_ПАПКИ"
```

Ожидается JSON с полем `layout` и списком `createdFolders` (при первом запуске появятся записи о создании `_lena/...`).

Дополнительно:

```powershell
node src/cli.js drive workspace-layout "ВАШ_ID_ПАПКИ"
node src/cli.js drive agent-bundle "ВАШ_ID_ПАПКИ"
```

Если видите ошибку про сертификат при `npm install`, см. раздел про TLS в [GOOGLE_DRIVE.md](GOOGLE_DRIVE.md).

## Что происходит «под капотом» технически

1. Node загружает JSON ключа.
2. Скрипт строит **JWT** (подпись `RS256` через `node:crypto`), обменивает его на **access token** у `https://oauth2.googleapis.com/token`.
3. Запросы идут по **HTTPS** к **Google Drive API** (`https://www.googleapis.com/drive/v3/...`) с заголовком `Authorization: Bearer …`.

Отдельно поднимать Cloud Run / VM **не обязательно** для локального CLI. Пакет **`googleapis` не используется** — меньше зависимостей и нет необходимости в `npm install` ради Диска.

---

## Частые проблемы

| Симптом | Что проверить |
|--------|----------------|
| `403` / Access not configured | Drive API не включён (шаг 4) или папка не расшарена на `client_email` (шаг 9). |
| `404` на файлах | Файл не в папке, доступной аккаунту; неверный `fileId`. |
| `npm install` и TLS | Актуально только если снова появятся npm-зависимости. Для `drive` они не нужны. |
