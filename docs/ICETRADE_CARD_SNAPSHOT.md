# Карточка IceTrade · шаблон снимка импорта

Опорный пример в репозитории: **`https://icetrade.by/tenders/all/view/1336510`** (тот же URL-паттерн `…/tenders/all/view/<view_id>`; компоновка блоков на других закупках близка).

Один URL вида `https://icetrade.by/tenders/all/view/<view_id>` после рендера (часто Playwright) даёт **повторяющуюся логическую структуру** страницы: блоки заголовков, таблицы «подпись ячейки → значение», таблица лотов, список файлов, хронология событий. Разметка (Vue/DOM) между закупками близка; отличаться могут формулировки строк и набор полей (часть полей пустая).

## Логические блоки (как на площадке)

1. **Идентификация** — строка вида «Процедура закупки № …», тип процедуры («Запрос ценовых предложений», аукцион и т.д.).
2. **Общая информация** — отрасль, краткое описание предмета.
3. **Заказчик / организатор** — кто проводит закупку, полное наименование и адрес, УНП, контакты (ФИО, телефоны, e-mail).
4. **Основная информация по процедуре** — даты (размещение приглашения, окончание приёма), ориентировочная стоимость, требования к участникам, квалификация, иные сведения.
5. **Конкурсные документы** — сроки/место/порядок выдачи документов, цена комплекта документов.
6. **Подача предложений** — место и порядок сдачи предложений (в т.ч. адрес, упаковка конверта).
7. **Лоты** — номер лота, предмет, количество/стоимость, статус; раскрывающиеся поля: срок поставки, место, источник финансирования, обеспечение, код ОКРБ.
8. **Документы** — ссылки на `.doc` / `.docx` / `.pdf` и т.п.
9. **События в хронологическом порядке** — дата/время, описание (в т.ч. отмена).

## Файл `inputs/icetrade-import-snapshot.json`

- **`schemaVersion`** — версия схемы снимка (см. код `importPageMeta.js`).
- **`kind`** — `icetrade.import.snapshot`.
- **`pageUrl`**, **`viewId`**, **`cardFetchVia`**, **`fetchedAt`** — происхождение и время.
- **`title`**, **`procedure`**, **`titleSources`**, **`headings`** — заголовки и идентификация процедуры.
- **`phones`**, **`emails`**, **`priceHints`** — агрегаты по тексту карточки.
- **`labeledFields`** — «как на сайте»: ключ = подпись из таблицы (RU), значение = текст.
- **`structured`** — **тот же контент**, разложенный в **фиксированные ключи** для кода и интеграций (не зависят от точного совпадения строки подписи при мелких правках площадки).
- **`documentLinks`** — пары `{ name, href }` для вложений с расширениями файлов.
- **`events`** — хронология (`date`, `text`, `severity`).
- **`warnings`** — эвристики (короткий HTML, нет таблиц в DOM и т.д.).

При отсутствии поля в HTML соответствующее значение в **`structured`** / массивах — `null` или пустой массив.

## Стабильные ключи `structured`

| Раздел JSON | Смысл |
|-------------|--------|
| `structured.general` | отрасль, краткое описание предмета |
| `structured.customer` | кто проводит закупку; реквизиты заказчика; контактные лица |
| `structured.procedure` | даты, ориентировочная стоимость, требования |
| `structured.competitiveDocuments` | выдача конкурсной документации, цена комплекта |
| `structured.bids` | приём предложений |
| `structured.lotDetails` | детали лота: срок/место поставки, финансирование, обеспечение, ОКРБ |

Точные имена полей — в JSDoc у `buildStructuredIceTradeCard` в **`src/icetrade/importPageMeta.js`**.
